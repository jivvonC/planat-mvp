# 00_ONBOARDING.md

# Screen

Onboarding

---

# Purpose

Introduce the philosophy of Planat, personalize the experience, explain the core features, and collect the minimum information required before entering the main service.

The onboarding should feel calm, welcoming, and personal.

Users should feel as though they are beginning a journey with their plants rather than setting up another productivity app.

# UX Principles

The onboarding should never feel like account setup.

Instead,

it should feel like slowly entering the world of Planat.

Every screen should answer one question:

"Why does this feature help me build a better relationship with my plants?"

---

# Cursor Notes

Prioritize emotion before explanation.

Avoid showing multiple features on one screen.

Use generous spacing.

One illustration per screen.

One clear message per screen.

Animations should be slow and calm.

The onboarding should feel like reading the first few pages of _The Little Prince_, not configuring an application.
